# Murmur — AI 自演化文字世界

> 打开页面，一个小世界里的 AI 角色们在自动生活、对话、博弈。你只是窥屏的观察者。关掉页面，世界暂停。

---

## 1. 产品概述

### 1.1 一句话定义

Murmur 是一个 AI 多角色自动叙事平台。用户创建或选择一个世界模板，启动一个独立的世界实例，AI 角色们在其中自主对话和行动。用户作为观察者旁观故事展开，无需任何交互输入。

### 1.2 核心理念

- **观察而非参与** — 用户不输入任何指令，纯粹旁观 AI 世界的自然演化
- **观察者驱动** — 有人观看时世界运转，无人观看时世界冻结，零空转成本
- **同源异变** — 同一个世界模板可以启动无限个实例，每个实例因模型随机性产生完全不同的剧情
- **故事沉淀** — 每一局世界的历史可以一键转化为可阅读、可分享的小说

### 1.3 目标用户

- 对 AI 能力好奇但不想学技术的普通用户
- 喜欢看剧/追番/读网文的内容消费者
- 想用 AI 做创意写作素材的作者
- 对多 Agent 系统感兴趣的开发者和研究者

## 2. 核心概念

### 2.1 术语定义

| 术语 | 定义 |
|------|------|
| **世界模板 (World Template)** | 一组预定义的世界设定 + 角色卡，是创建世界实例的蓝图 |
| **世界实例 (World Instance)** | 从模板实例化的一个独立运行的世界，拥有独立的状态和历史 |
| **角色 (Character)** | 世界中的一个 AI Agent，拥有独立的人格、记忆和行为逻辑 |
| **导演 (Director / DM)** | 特殊的 Agent，不作为角色出现，负责叙事、节奏控制和事件注入 |
| **回合 (Turn)** | 世界推进的最小单位，每回合由导演决定一个角色行动或注入一个事件 |
| **观察者 (Observer)** | 正在观看某个世界实例的用户 |
| **冻结 (Frozen)** | 世界实例无观察者时的暂停状态 |

### 2.2 世界生命周期

```
模板 → 用户点击「创建新世界」→ 实例化 → 冻结状态
  → 首个观察者进入 → 世界开始运转 → 回合推进
  → 所有观察者离开 → 世界冻结
  → 观察者再次进入 → 从冻结点继续
  → 达到结局条件 / 回合上限 → 世界终结 → 可写成小说
```

## 3. 世界模板系统

### 3.1 模板结构

每个世界模板包含以下字段：

```yaml
id: midnight-tavern
name: "深夜酒馆"
genre: "悬疑推理"
description: "暴风雪困住了酒馆里的五个人，其中一个是昨晚凶案的凶手。"
cover_emoji: "🏮"

setting:
  background: |
    大雪封山，通往镇外的唯一道路被阻断。
    这间破旧的酒馆里，五个素不相识的人被迫共度长夜。
    昨晚，镇上的当铺老板被人杀死在自己的店里。
    凶手就在这五个人之中。
  tone: "紧张、压抑、偶有黑色幽默"
  era: "当代中国小镇"

characters:
  - id: laozhang
    name: "老张"
    role: "酒馆老板"
    personality: "表面热情周到，实际精明算计"
    secret: "三年前帮逃犯藏过身，那人至今未被抓获"
    relationships:
      ali: "好奇但警惕，觉得她不简单"
      wangliu: "厌恶，拖欠三个月酒钱"
      lishen: "老邻居，基本信任"
      xiaomei: "同情，觉得她在逃避什么"
    speech_style: "口语化，爱用歇后语，偶尔流露生意人的算计"

  - id: ali
    name: "阿离"
    role: "神秘旅人"
    personality: "冷静、观察力强、话不多但句句有深意"
    secret: "她是死者的女儿，回来调查真相"
    # ... 其他角色类似

director:
  pacing: |
    前 10 回合：角色互相试探，建立初步关系
    10-30 回合：线索逐步浮现，冲突升级
    30-50 回合：关键秘密暴露，走向高潮
    50+ 回合：引导结局
  event_triggers:
    - condition: "对话陷入重复或停滞"
      action: "注入突发事件（停电、有人发现新线索、外面传来声响）"
    - condition: "某角色的秘密相关线索被多次暗示"
      action: "安排秘密被意外发现的情节"
  ending_conditions:
    - "凶手身份被正确指认"
    - "凶手成功转移嫌疑，另一个人被冤枉"
    - "达到 80 回合仍无定论，暴风雪停了，警察到来"

max_turns: 80
```

## 4. Agent 系统

### 4.1 角色 Agent

每个角色是一个独立的 AI Agent，拥有：

**System Prompt 结构：**

```
你是 {name}，{role}。

【性格】{personality}

【秘密】你知道一个秘密：{secret}。你不会主动透露，除非被逼到绝境或有足够的理由。

【关系】
- 对 {character_a}：{relationship_a}
- 对 {character_b}：{relationship_b}

【说话风格】{speech_style}

【规则】
- 你只能说出你的角色知道的信息
- 你的回复应为 1-3 句自然对话，不要长篇大论
- 根据当前情境自然反应，可以撒谎、回避、反问
- 不要打破第四面墙，不要提及自己是 AI
```

**私有记忆：** 每个角色维护一个私有上下文窗口，只包含：
- 自己的角色设定
- 自己经历过的公开对话
- 自己做过的私下行动（其他角色不可见）
- 导演给的私下提示（如"你注意到阿离在偷看老张的抽屉"）

### 4.2 导演 Agent (Director / DM)

导演不是角色，是世界的"上帝视角"控制器：

**System Prompt 结构：**

```
你是这个世界的导演，负责推进叙事。

【世界设定】{setting.background}

【所有角色及其秘密】（导演知道一切）

【节奏指南】{director.pacing}

【你的职责】
1. 决定下一步谁说话（输出 next_speaker）
2. 必要时生成旁白描写（环境、氛围、角色动作）
3. 在对话停滞时注入突发事件
4. 追踪秘密的暴露进度，在合适时机安排揭露
5. 判断是否达到结局条件

【输出格式】
{
  "type": "narration" | "character_turn" | "event" | "ending",
  "next_speaker": "character_id",  // type 为 character_turn 时
  "narration": "环境描写或旁白",  // type 为 narration 或 event 时
  "event_description": "突发事件描述",  // type 为 event 时
  "ending_summary": "结局总结"  // type 为 ending 时
}
```

**导演的决策输入：** 每次决策时，导演看到完整的世界历史（所有公开对话 + 旁白 + 事件），据此判断节奏和下一步行动。

### 4.3 回合推进流程

```
每个回合：
  1. 导演 Agent 接收世界历史 → 输出下一步指令
  2. 根据指令类型：
     a. narration → 生成旁白，推送给所有观察者
     b. character_turn → 被指定的角色 Agent 生成对话
     c. event → 生成突发事件描写，所有角色下一轮需对此反应
     d. ending → 世界终结，生成结局总结
  3. 将本回合内容写入数据库
  4. 通过 WebSocket 推送给所有观察者
  5. 等待间隔时间 → 进入下一回合
```

### 4.4 消息可见性规则

| 消息类型 | 角色可见性 | 观察者可见性 |
|----------|-----------|-------------|
| 公开对话 | 所有角色 | 可见 |
| 旁白/事件 | 所有角色 | 可见 |
| 角色内心独白 | 仅本角色 | 可见（以特殊样式显示）|
| 导演对角色的私下提示 | 仅目标角色 | 不可见 |
| 角色的秘密 | 仅本角色 | 秘密揭露后可见 |

## 5. 观察者系统

### 5.1 观察者触发机制

```
WebSocket 连接管理：
  - 用户打开世界页面 → 建立 WebSocket → 观察者计数 +1
  - 观察者计数从 0 变为 1 → 启动世界时钟
  - 用户关闭页面 → WebSocket 断开 → 观察者计数 -1
  - 观察者计数从 1 变为 0 → 停止世界时钟（冻结）

心跳机制：
  - 客户端每 30 秒发送心跳
  - 60 秒无心跳视为断开
```

### 5.2 观察者体验

**进入世界时：**
- 加载世界基本信息（模板名、角色列表、当前回合数）
- 加载最近 20 条历史消息（可上滑加载更多）
- 从当前进度开始实时接收新消息

**观看过程中：**
- 新消息以打字机效果逐字出现
- 可调节推进速度（慢 15s / 正常 8s / 快 3s）
- 可暂停当前世界（仅暂停自己的推送，不影响世界运转）
- 可点击角色名查看公开信息

**离开后重新进入：**
- 从上次离开的位置继续，中间的历史可以回看

### 5.3 速度控制

```
观察者设置的速度只影响推送间隔：
  - 慢速：每 15 秒推进一个回合
  - 正常：每 8 秒推进一个回合
  - 快速：每 3 秒推进一个回合

当多个观察者观看同一个世界实例时：
  - 世界按最快观察者的速度推进
  - 较慢的观察者可能会积压消息，自动以更快速度追赶
```

## 6. 小说生成系统

### 6.1 触发条件

- 世界终结后（达到结局条件或回合上限）
- 用户手动点击「写成小说」（可在世界运行中的任何时刻，基于已有历史生成）

### 6.2 生成流程

```
第一步：剧情提炼
  输入：完整的世界历史（所有回合的对话、旁白、事件）
  输出：结构化的剧情大纲
    - 起因 → 发展阶段 → 转折点 → 高潮 → 结局
    - 核心冲突线
    - 各角色的故事弧

第二步：风格选择（用户选择）
  - 严肃文学风：注重环境描写和心理刻画
  - 网文爽文风：节奏紧凑，对话为主，冲突感强
  - 吐槽解说风：以旁观者视角吐槽角色行为，轻松幽默

第三步：小说生成
  输入：剧情大纲 + 原始对话记录 + 风格指令
  输出：带章节结构的完整小说文本（约 3000-8000 字）

第四步：排版输出
  - 网页阅读版（带章节导航）
  - 分享长图（精美排版 + 世界信息水印）
  - Markdown 文件下载
```

### 6.3 风格示例

**同一段对话的三种风格输出：**

原始对话：
> 老张："你已经是第三天来这儿了，还不说你到底找谁？"
> 阿离："找谁不重要，重要的是有人不想让我找到。"
> 王六："我昨晚看见你在码头和一个穿黑衣的人说话。"

严肃文学风：
> 张老板擦杯子的动作顿了一下。三天了，这个女人每晚都坐在靠窗的位置，只点一杯最便宜的米酒，目光却在每个人身上停留。他终于按捺不住地开了口。
> 女人放下酒杯，嘴角浮起一丝意味不明的笑。"找谁不重要。"她的声音不大，却让吧台另一头的王六竖起了耳朵。"重要的是，有人不想让我找到。"
> 赌桌上的王六突然插了一嘴，带着几分酒意和几分精明——这两样东西在他身上总是分不清楚。

网文爽文风：
> "你到底找谁？"老张终于忍不住了。
> 阿离淡淡一笑，放下酒杯："有人不想让我找到。"
> 此话一出，酒馆瞬间安静了几秒。
> "我昨晚看见你了。"王六冷不丁冒出一句，"码头，穿黑衣的那个人，是谁？"
> 空气突然凝固。

吐槽解说风：
> 老张这个人吧，忍了三天终于破防了。你说你一个开酒馆的，人家来消费你管人家找谁呢？
> 但阿离的回答更绝——"有人不想让我找到"——好家伙，这台词说出来，整个酒馆的氛围瞬间从《乡村爱情》切换到了《无间道》。
> 然后王六这个赌鬼就不合时宜地蹦出来了。兄弟，你一个欠了三个月酒钱的人，有什么资格质问别人？

## 7. UGC 世界编辑器（v2/v3）

### 7.1 创建流程（面向非技术用户）

**引导式向导，共 4 步：**

```
Step 1：世界基础信息
  - 世界名称
  - 类型标签（悬疑/日常/奇幻/武侠/科幻/喜剧）
  - 一句话描述
  - 背景设定（200 字以内，或让 AI 帮你扩写）

Step 2：创建角色（2-6 个）
  每个角色填写：
  - 名字
  - 身份/职业
  - 性格（可从预设标签选：冷静/暴躁/腹黑/天真/...）
  - 一个秘密（可选）
  - 和其他角色的关系（简单选择：信任/敌对/暧昧/陌生）
  - 说话风格（可选，不填则 AI 根据性格自动生成）

Step 3：世界规则
  - 有无终局（开放式 / 有明确结局条件）
  - 最大回合数（20/50/80/不限）
  - 基调（严肃/轻松/荒诞）

Step 4：预览与发布
  - AI 根据填写内容自动生成完整的角色卡和导演指令
  - 试运行 5 个回合预览效果
  - 满意后发布到世界市场
```

### 7.2 世界市场

- 所有用户发布的世界模板展示在公共市场
- 按热度（启动实例数）、评分、最新排序
- 用户可以收藏、评分、基于别人的模板「Fork」修改

### 9.2 数据模型

```sql
-- 世界模板
CREATE TABLE world_templates (
  id          TEXT PRIMARY KEY,
  name        TEXT NOT NULL,
  genre       TEXT,
  description TEXT,
  config      JSONB NOT NULL,     -- 完整的模板 YAML 转 JSON
  author_id   TEXT,               -- UGC 作者（v2）
  play_count  INTEGER DEFAULT 0,  -- 被实例化次数
  created_at  TIMESTAMP DEFAULT NOW()
);

-- 世界实例
CREATE TABLE world_instances (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  template_id   TEXT REFERENCES world_templates(id),
  status        TEXT DEFAULT 'frozen',  -- frozen / running / ended
  current_turn  INTEGER DEFAULT 0,
  observer_count INTEGER DEFAULT 0,
  world_state   JSONB,                  -- 当前角色状态快照
  created_at    TIMESTAMP DEFAULT NOW(),
  ended_at      TIMESTAMP
);

-- 世界事件（每一回合的记录）
CREATE TABLE world_events (
  id           SERIAL PRIMARY KEY,
  instance_id  UUID REFERENCES world_instances(id),
  turn         INTEGER NOT NULL,
  event_type   TEXT NOT NULL,           -- narration / dialogue / event / ending
  character_id TEXT,                    -- 发言角色（dialogue 时）
  content      TEXT NOT NULL,           -- 对话或旁白内容
  metadata     JSONB,                  -- 标签（反转/结盟等）、情绪、关系变化
  created_at   TIMESTAMP DEFAULT NOW()
);

-- 生成的小说
CREATE TABLE novels (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  instance_id  UUID REFERENCES world_instances(id),
  style        TEXT NOT NULL,           -- literary / webnovel / commentary
  content      TEXT NOT NULL,
  word_count   INTEGER,
  created_at   TIMESTAMP DEFAULT NOW()
);
```

### 9.3 Agent 调用流程（伪代码）

```python
async def run_turn(instance_id):
    # 1. 加载世界状态和历史
    instance = db.get_instance(instance_id)
    history = db.get_events(instance_id, limit=50)  # 最近 50 条作为上下文

    # 2. 导演决策
    director_prompt = build_director_prompt(instance.template, history)
    director_response = await llm.call(
        model="deepseek-v3",
        system=director_prompt,
        messages=format_history(history),
        temperature=0.9
    )
    decision = parse_director_output(director_response)

    # 3. 根据决策类型执行
    if decision.type == "character_turn":
        character = instance.template.characters[decision.next_speaker]
        char_prompt = build_character_prompt(character, history)
        dialogue = await llm.call(
            model="deepseek-v3",
            system=char_prompt,
            messages=format_character_history(history, character.id),
            temperature=0.95
        )
        event = create_dialogue_event(character, dialogue)

    elif decision.type == "narration":
        event = create_narration_event(decision.narration)

    elif decision.type == "event":
        event = create_event_event(decision.event_description)

    elif decision.type == "ending":
        event = create_ending_event(decision.ending_summary)
        instance.status = "ended"

    # 4. 持久化
    db.save_event(instance_id, event)
    instance.current_turn += 1
    db.update_instance(instance)

    # 5. 推送给观察者
    websocket.broadcast(instance_id, event)
```

## 13. 风险与应对

| 风险 | 影响 | 应对策略 |
|------|------|---------|
| AI 生成内容不当（暴力/色情） | 用户体验差，合规风险 | 导演 prompt 中明确禁止规则；输出层关键词过滤 |
| 对话陷入无意义循环 | 故事无聊，用户流失 | 导演设置循环检测逻辑，自动注入事件打破僵局 |
| DeepSeek API 不稳定 | 世界卡顿 | 备选模型（通义千问/豆包），自动 failover |
| 角色"出戏"（打破第四面墙） | 沉浸感破坏 | prompt 强化规则 + 输出后处理检测 |
| 单世界观察者过多导致消息风暴 | 服务器压力 | 同一世界只跑一份 Agent，广播给所有观察者 |
| 免费用户滥用创建大量世界 | 资源浪费 | 限制每人同时运行的世界实例数（如 3 个） |
